다음은 **현재 화면을 기준으로 한 재설계안(구조·컴포넌트·UX까지 포함한 1차 설계안)**입니다.
시각적 감성은 유지하되, **제품 완성도와 사용 흐름**을 기준으로 재구성합니다.

---

# OUTBRAIN 일기 작성 화면 재설계안

## 1. 설계 목표

1. “무엇을 해야 할지 바로 알 수 있는 화면”으로 만든다.
2. 쓰기 → 저장 → 회고 흐름이 자연스럽게 이어지도록 한다.
3. 기록 행위에 집중하도록 시선 분산 요소를 줄인다.

---

## 2. 전체 레이아웃 재설계

### 기존

* 좌측: 작성
* 우측: 캘린더 + 카드 (동일한 시각적 비중)

### 개선

**Primary / Secondary 영역 명확 분리**

```
┌───────────────────────────────┐
│ Header (검색 | 쓰기 | 읽기 | 인사이트)
├───────────────────────────────┤
│                               │
│  Main Writing Area (70%)       │  Side Panel (30%)
│                               │
│                               │
└───────────────────────────────┘
```

---

## 3. Header 재설계

### 변경 포인트

* “쓰기” 상태를 명확히 인지 가능하게
* 저장은 헤더 고정 액션으로 승격

### 구조

* 좌측: OUTBRAIN 로고
* 중앙: 탭 (검색 / 쓰기 / 읽기 / 인사이트)
* 우측: **저장 버튼 (Primary CTA)**

저장 버튼 상태

* 비활성: 연한 피치
* 활성: 진한 피치
* 저장 중: 로딩
* 완료: 체크 + “저장됨” 1.5초 표시

---

## 4. 메인 작성 영역 (Primary)

### 4-1. 상단 질문 영역

**현재**

* 질문 + 설명

**개선**

* 질문을 화면의 “앵커”로 사용

```
오늘 하루 어땠어?
[오늘의 생각을 자유롭게 남겨보세요]
```

* 질문: 32px Bold
* 설명: 14px Regular, 대비 약하게

---

### 4-2. 제목 / 날짜 입력

#### 재배치

* 제목과 날짜를 하나의 “메타 카드”로 묶음

```
[ 제목 입력 __________________ ]   [ 📅 2026.01.31 ]
```

* 카드 형태
* 배경: white
* radius: 16px

---

### 4-3. 기분 선택 UI 재설계

#### 기존 문제

* 선택 여부가 약함
* 감정의 “강도”가 드러나지 않음

#### 개선 구조

* 섹션 타이틀 강조

  ```
  오늘의 기분
  ```

* 아이콘 버튼 형태 유지

* 선택 시:

  * 배경 채움
  * 아이콘 확대 (1.08x)
  * 라벨 색 강조

#### 감정 구성 제안 (7단계)

1. 매우 좋음
2. 기쁨
3. 평온
4. 보통
5. 피곤
6. 슬픔
7. 힘듦

---

### 4-4. 에디터 영역 재설계

#### 목표

* “막막한 빈 화면” 제거
* 자연스러운 첫 문장 유도

#### 초기 상태

* Placeholder 문장 표시

```
오늘 기억에 남는 장면은 무엇이었나요?
누군가와 나눈 대화, 감정이 움직인 순간,
사소해 보여도 괜찮아요.
```

#### 에디터 UX

* 툴바 대비 낮춤
* 본문 영역 시각적 집중
* 첫 입력 시 Placeholder 자연스럽게 사라짐

---

## 5. 사이드 패널 재설계 (Secondary)

### 5-1. 날짜 카드 (캘린더 축소형)

* 월 캘린더 유지
* 크기 축소
* “선택한 날짜” 강조만 유지

---

### 5-2. 오늘의 생각 프롬프트 카드

#### 현재

* 존재감 약함

#### 개선

* 카드 컴포넌트로 승격

```
💬 오늘의 질문
“오늘 예상하지 못해 미소 지었던 순간은?”
```

* 클릭 시:

  * 에디터 상단에 자동 삽입
  * 커서 이동

---

### 5-3. 인사이트 진입

* 버튼 형태로 명확히
* “이 기록에서 인사이트 보기”
* 저장된 이후에만 활성화

---

## 6. 인터랙션 흐름 정리

1. 화면 진입
2. 질문 → 제목 → 기분 → 글쓰기
3. 저장 버튼 활성화
4. 저장 완료 피드백
5. 인사이트 또는 다음 기록 유도

---

## 7. 디자인 톤 유지 가이드

* 색상: 기존 피치·소프트 파스텔 유지
* 대비만 살짝 강화
* “따뜻하지만 명확한 UI” 지향
